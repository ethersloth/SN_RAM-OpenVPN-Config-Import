Welcome to the Crimson Remote Link Walkthrough

Crimson Remote Link will allow downloading and extracting Crimson databases through this device's cellular connection.
Any Crimson database access supported from the Crimson Link menu can be supported remotely through this feature.

Make sure to configure your Crimson database to enable IP Download. Please consult your Crimson manual's section Download Settings for more information.

Once your Crimson device is setup on the same local Ethernet network as this router, we can access it and forward requests from the cellular environment. You can check your Crimson Ethernet settings in the Communications-Network, Ethernet tab.

Confirm that this is in the same subnet as this unit's Ethernet settings, Network -> Interfaces -> Eth0 or Eth1, depending on your cabling setup.

Security

Since this remote unit could be reconfigured or reloaded remotely, we may want to enable some protection against this access. Once the Crimson Remote Link is activated, it will only stay open for a few minutes. New attempts after this timeout will be rejected, but current connections will not be prematurely terminated.

Access can also be restricted to a specific IP address that you provide. If you will be accessing the remote link from the same PC that you are using right now, we can detect the IP address you are using now and provide that as the only allowed IP address.

Crimson SMS API Configuration

When you enable the SMS API, a Crimson application can use this cellular gateway to send and receive SMS messages. Search for Crimson RAM SMS API in the Red Lion knowledge base.

NOTE: If using the "and Serial" option, it will prevent other serial port activity such as Modbus. Choose "TCP Only" if you are also using protocol conversion on the serial port.

SN Proxy Settings

When the proxy is enabled, these settings can offer improved performance of cellular access to the web interface of a Crimson product. Screen updates over cellular can be more efficient and additional security can be achived with an HTTPS (Secure SSL) connection and an extra user login step.

Cellular carriers often block port 80, so we recommend an alternate access port for browsers connecting to the cellular IP address. The status screen will show the full browser link to use.
