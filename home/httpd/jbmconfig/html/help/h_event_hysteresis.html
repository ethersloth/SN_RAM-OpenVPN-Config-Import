Description

Hysteresis is used to prevent an event from oscillating between the on and off states when the process is near the alarm condition. For example, for an absolute high alarm, the event will become active when the input exceeds the alarm's value, but will only deactivate when the input falls below the value by an amount greater than or equal to the alarm's hysteresis. Remember that the property always acts to maintain an alarm once the alarm is activated, and not to modify the point at which the activation occurs. If configured, hysteresis and the deactivation delay must both be satisfied for the alarm to be cleared. Default: 0 to disable.
