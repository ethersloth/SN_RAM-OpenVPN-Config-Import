Description

Specify the number of lost packets that are acceptable before the link is considered unavailable.

Note: The value must be less than the number of test packets set via Test Packets to Send

Example

If Test Packets to Send is set to 5, and Allowable Packet Loss is set to 3, then 2 pings of the 5 sent out must have replies for connectivity to be declared successful. If only 1 ping reply is received, then a failure to that host will be declared.
