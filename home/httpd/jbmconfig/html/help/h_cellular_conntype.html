Description

In Persistent mode, the unit will always attempt to maintain a cellular connection with the carrier's network. In On-Demand mode, the connection to the cellular network will only be attempted when packets are destined to leave the wireless interface. In addition, after a period of Idle Time, the connection will terminate.

Persistent mode has the potential to use more bandwidth in a limited data plan meant for fallback. For this reason, On-Demand is commonly chosen for an environment using an ethernet connection as a primary interface, while IP Fallback is used to bring up the secondary wireless connection as a backup. Modern Cellular connections use idle states in on-board radios to often negate the need for an On-Demand option.
